Problem Solving Served Two Ways

Blocked
What was the problem? During Sprint 4 I had come across an exercise where the code would produce the output but had just one very minor issue with it that didn`t let it pass. No matter how I wrote the code it would still have the issue.
What problem-solving techniques did you use? I used multiple techniques:
  • Googling
  • Brute force of just jamming in the different possible ways I think it might work
  • Taking time away from the problem
  • Trying to debug the code to find issues
  • Console logging
How did you feel throughout the process? The feeling was pure frustration, I just couldn’t see why things weren’t working
What did you learn? I learned many different methods for strings and arrays as a result of googling and slowing inching one step closer to knowing how to debug like a coder

Elegant
What was the problem? Elegant problem solving isn’t one of my strong suits. I believe that one of the only elegant problem solving events is when the code has an error due to a simple syntax or spelling mistake
What problem-solving techniques did you use? As the only times i have elegantly problem solved was when the problem was fairly simple I only use:
  • Console logging
  • Reading/checking the code over
  • Taking time away from the problem
How did you feel throughout the process? Generally this process results in a much calmer life, however, still annoying that I messed up something so simple
What did you learn? Makes me realise I need to be more careful and try and avoid little errors like that

A time where I was reluctant to ask

I think throughout the foundation I have been reluctant to ask for help a lot. I think it's mainly out of stubbornness and wanting to find the answers for myself. This is generally because I usually finish with extra time that allows me to take some time and problem solve for myself. I think it also stems from the fact I learn by doing rather than being told.
